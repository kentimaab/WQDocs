# translate-sv

## Objective

A one-off Python script (`translate_sv.py`) that batch-translates documentation pages from English to Swedish. Source truth is `docs/`; Swedish output lives in `docs-sv/`. Pages in `docs-sv/` currently contain a placeholder body ("Översättning kommer") — this script replaces those placeholders with real Swedish translations generated by Claude. Pages that already have real Swedish content are left untouched.

## Requirements

1. The script is a standalone Python file: `translate_sv.py` at the repo root.
2. If the `anthropic` package is not installed, the script installs it via `pip` before proceeding.
3. If `ANTHROPIC_API_KEY` is not set in the environment, the script exits immediately with a clear error message explaining what to set.
4. For each `.md` file in `docs-sv/`, the script checks whether the body (everything after the frontmatter block) is a placeholder. The actual placeholder body format is a single heading line followed by `Översättning kommer` and nothing else — e.g. `# Some Title\n\nÖversättning kommer`. The check: strip the body of leading/trailing whitespace, remove the first line if it starts with `#`, strip again, and check whether what remains equals `Översättning kommer`. If the body is not a placeholder — i.e. real Swedish content already exists — skip the file without modification.
5. For files that do need translation, the script reads the matching source file from `docs/` using the same relative path (e.g. `docs-sv/foo/bar.md` → `docs/foo/bar.md`). If no matching source file exists, log a warning and skip.
6. The script translates to Swedish using Claude (`claude-sonnet-4-6`) via the `anthropic` Python SDK.
7. **Translate:** body text, all headings (`#`, `##`, etc.), frontmatter `title` value, frontmatter `description` value.
8. **Do NOT translate:** code blocks (fenced ` ``` ` blocks and indented code), inline backtick-wrapped content, product names (WideQuick, MOD, BMS, WWT, Kentima), UI label names, frontmatter keys (e.g. `title:`, `description:`, `slug:`), file paths, HTML tags and their attributes (class names, style values, href/src values), MkDocs Material syntax (icon shortcodes such as `:material-view-grid-outline:`, admonition markers, grid card markers), and the URL portion of Markdown links (translate only the visible link text, not the URL in parentheses).
9. The translated content is written back to the same `docs-sv/` file, preserving frontmatter structure and all keys — only the `title`, `description` values and body are changed.
10. The script prints progress to stdout for each file: filename, status (`skipped` / `translated` / `error`).

**Out of scope:**
- DeepL or any translation provider other than Claude.
- Re-running incrementally as new files are added (one-off use).
- Any changes to files in `docs/`.
- Translating filenames or directory names.

**Nice to have:**
- Dry-run mode (`--dry-run`) that prints what would be translated without writing files.

## Constraints

- Python 3, no external dependencies beyond `anthropic` (auto-installed).
- Uses `claude-sonnet-4-6` model.
- `ANTHROPIC_API_KEY` must be available as an environment variable at runtime.
- Must not modify files in `docs/`.
- Must preserve all frontmatter keys and values except `title` and `description`.

## Edge cases

- **File in `docs-sv/` has no matching file in `docs/`:** log a warning (`WARNING: no source for <path>, skipping`) and skip.
- **Frontmatter has no `description` key:** translate only `title` and body; do not add a `description` key.
- **Body contains a mix of placeholder and real content:** treat any body that is not exactly "Översättning kommer" (whitespace-trimmed) as already translated — skip it.
- **Code blocks inside body:** pass to Claude with explicit instruction not to translate fenced or indented code blocks, HTML tags/attributes, MkDocs icon shortcodes, or the URL part of Markdown links.
- **Product names in running text:** Claude is instructed via system prompt to leave WideQuick, MOD, BMS, WWT, and Kentima unchanged wherever they appear.
- **Claude API error on a file:** log the error, skip that file, continue with the rest.

## Definition of done

- [ ] `translate_sv.py` exists at repo root and runs with `python translate_sv.py`.
- [ ] Running without `ANTHROPIC_API_KEY` set prints a clear error and exits non-zero.
- [ ] If `anthropic` is not installed, the script installs it and continues without manual intervention.
- [ ] A file whose body is already real Swedish content is not modified.
- [ ] A file whose body is "Översättning kommer" is translated: body, headings, frontmatter `title`, frontmatter `description` are in Swedish.
- [ ] Code blocks in the translated output are unchanged from the source.
- [ ] Product names (WideQuick, MOD, BMS, WWT, Kentima) are unchanged in the translated output.
- [ ] A `docs-sv/` file with no matching `docs/` counterpart triggers a warning, not a crash.
- [ ] A Claude API failure on one file does not abort the rest of the batch.
- [ ] All frontmatter keys and non-translated values are preserved exactly.

## Assumptions

- The `docs-sv/` directory structure mirrors `docs/` exactly (same relative paths).
- "Översättning kommer" is the only placeholder string to detect; no other variants.
- The `title` field in `docs-sv/` frontmatter is inconsistent — some files have a real Swedish title, others have raw slugs like `configuring` or `index`. The script always re-translates `title` from the English source in `docs/`, overwriting whatever is currently in `docs-sv/`.
- UI label names that should not be translated will be recognizable in context by Claude (no explicit list is provided — Claude is instructed generally to leave UI labels, menu names, and button labels untranslated).
